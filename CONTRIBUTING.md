# Contributing

Thanks for your interest in contributing to this project! We welcome contributions of all kinds — bug reports, feature requests, documentation improvements, tests, and code changes.

## Code of Conduct
By participating in this project you agree to follow the project's Code of Conduct. See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) for details.

## How to Contribute

- **Reporting bugs:** Open an issue with a clear title and steps to reproduce. Include environment details and relevant logs or stack traces.
- **Feature requests:** Open an issue describing the problem, the use case, and any proposed design or API suggestions.
- **Discuss first for large changes:** For large or potentially breaking changes, open an issue to discuss the design before working on a PR.

## Pull Request Process

1. Fork the repository and create a branch named `fix/short-description` or `feat/short-description`.
2. Write tests for any new behavior or bug fix.
3. Ensure the test suite passes locally.
4. Run a linter/formatter if applicable.
5. Submit a pull request with a clear title and description explaining the change and why it is needed.

PRs should be small and focused. The maintainers will review, provide feedback, and request changes as needed.

## Coding Style

- Follow the existing code style and conventions used in the repository.  For Rust use the default code formatter (rustfmt).
- Keep functions and modules small and focused.
- Name things clearly and document non-obvious behavior.

## Commit Messages

Use clear, descriptive commit messages. Prefer the imperative present tense (e.g., "Add feature", "Fix bug in parser").

## Tests

- Add tests for new features and bug fixes.  We can't fix bugs without replay tests.
- Make sure existing tests pass before opening a PR.

If there are test or build instructions for this repository, include them in the PR description so reviewers can reproduce locally.

## Continuous Integration

This project may run CI for PRs. Fix any failures reported by CI before merging.

## Licensing

By contributing you agree that your contributions will be licensed under the project's license. See [LICENSE](LICENSE) for details.

## Questions

If you have questions about how to contribute, open an issue or contact the maintainers.

Thank you for helping improve this project!
